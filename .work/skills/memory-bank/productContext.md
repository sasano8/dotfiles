# Product Context

## なぜ存在するか / 解決する課題
- エージェント（Claude Code / Codex / Gemini）の設定・スキルがマシンごとにバラつき、
  手作業のコピーで陳腐化・差異が生じる問題を解消する。
- セッションをまたぐとエージェントの記憶がリセットされる前提で、**継続性**（Memory Bank）と
  **複数プロジェクトの俯瞰オーケストレーション**（supervisor）を仕組みとして持たせる。
- 個人ナレッジ（議事録など）の散逸を防ぎ、横断要約（docs-summary）で再利用可能にする。

## どう動くべきか
- 正本は git リポジトリ。`install.sh` で symlink を張るだけで各マシンに反映。
- スキルは 1 つの `SKILL.md` で複数エージェントが共通に読める（Agent Skills 標準）。
- SessionStart フックが「Memory Bank があれば読め」と自動案内する（無いプロジェクトでは no-op）。
- 破壊的・組織管理の設定は壊さない（非破壊リンク、FORCE で明示上書き）。

## UX / 利用者ゴール
- 利用者は本人（個人 dotfiles）。新マシンでの立ち上げが最小手順で済むこと。
- スキルの追加・改善が「リポジトリを編集 → commit」だけで全マシン・全エージェントへ波及すること。
- エージェントがリセット後も Memory Bank だけで作業文脈を取り戻せること。
